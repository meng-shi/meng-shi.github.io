Background

Objects such as toys and storybooks play an important role in childhood learning, not only helping teach the concept of physical interaction between objects but also feeding and shaping the imagination.

However, as technology becomes more pervasive, children are spending more of their play time on digital devices such as tablets.
Touchscreen interfaces often provide beneficial learning but the importance of learning through making has been long recognized.
Physical toy playing experience cannot be replaced.

Can we use a robust, low-cost technology to open up more learning through making scenarios for young kids?





Key criterias for the technology used on toys

  • Safe for children
  • Durable
  • Reliable
  • Affordable
  • Low-power
  • Scalable

    • Differential magnetic and resistive have limited scalability.
    Camera-based tracking suffers from occlusion.
    Active radios in all toys is an option but requires more powered objects and additional component costs.
    Likewise, ultrasonic emissions can be used to track objects, but each object would need a powered microphone or sound emitter.

    Thus, RFID is our top choice!

    Code maker

    We designed the code maker prototype to help children learn the basic concepts of programming.


    To control the firetruck, children place a series of RFID-embedded “command tiles” on the side of the truck, then press a button to cause the truck to execute the commands on the tiles.

    When the button is pressed, the microcontroller reads the antenna array, detecting the command tiles present, then activates actuators on the firetruck.

    Lego firetruck



    Code Maker includes an Android-based software user interface to allow children to compare tangible and software-based approaches to accomplish the same functionality in the robotic firetruck.

    We used Goggle Blockly in the creation of an application, which provides a simple user interface for laying out commands in a sequence.

    After children send the commands, the Android device, sends the command set to the firetruck, which compiles and executes the instructions through the actuators.

    Story maker

    We designed the Story maker prototype to help children learn by creating stories.


    We used “The Three Little Pigs” story as an example.

    This classic fable tells of three pigs who build three houses with three different materials. When the Big Bad Wolf comes to eat the pigs, he tries to blow down each house.

    In the prototype the wolf and the houses have RFID readers, and depending on the wolf’s proximity to a house, the system plays a part of the story on a computer screen, including associated text to help teach reading.
