Background


Laptop VS Mobile
Laptops: a non-interactive cuboid when its lid is closed.
Mobile: a convenience device that always connects with user.
When lid is closed, laptop has no interaction with human anymore.
How can we make better use of our laptop computer?

Can we add a second screen to the laptop, and using that screen to connect users while laptop's lid is closed?




Who is user?

Target User


Use PC laptop as primary productivity device

Use device in multiple locations (home, office, on-the-go) or have to walk to different conference rooms, parts of the office building throughout the day

Regularly multitask


Main Use cases


Laptop lid is closed and user is working nearby

Laptop lid is closed and user walks with device in hand

Laptop lid is open and its main screen is on.

Interview target users





UX/UI Design Challenges

Laptop with a secondary display is a novel concept with no prior art for determining


  • What content will users want on it?

  • How to present content (both UI and interaction design)?

  • How to ensure users' security and privacy?

  • How to efficiently collabrate with global teams to find a solution?




    • Content selesction

    Due to limited space of the second screen, it is important to limit what should be displayed on it and how to use other media such as sound to improve interaction.

    We used Eisenhower’s Urgent/Important frameworks to better understand types of content best suited for second screen.

    Considering that the second screen has a more direct connection to the user, work related Apps, which send important and urgent information are always our best choice, meeting notification/skype call need users’ attention immediately, the second screen can help to achieve that.

    Based on user study, we also found brief attentional demand like media controller is also welcome. User like it when lid is open and the main screen is fully covered by other application, a controller on the second screen which can be directly used to control media instead of interrupt the applications on the main screen is useful.

    Last but not least, users should be able to customize the second screen Apps to meet their specific needs: including selecting their interests, moving Apps in different order, altering colors, etc.





    How to simplify the selected APP for second screen use?

    Brainstorming, Analysis and Multiple prototyping iterations are the key elements.

    Brainstorming

    Brainstorming


    Main design principle established during iterations:

  • Minimize number of buttons
  • Short focus time
  • Use of voice vs touch
  • Multiple environment usages
  • Single task-simple function
  • Adapt to personal preference

    • Let's use Google Calendar as an example to explain how we simplify an application for a small edge screen.
    The complete calendar application usually contains multiple items, yearly view, monthly view, daily view, etc.
    For second screen use, we minimized the number of clicks needed and only show the most important upcoming events. Provide voice input and automatically notification at the same time.

    Collabration with teams around the world

    Challenge is not only from visual side, bring an design from scratch to reality needs tons of work from both hardware and software teams.
    The issues we met during the development include but not limited to: dead pixel on screen, limited API info, wake up method, etc.
    Organizing weekly design sync up meeting with stakeholders is important to push the design forward.
